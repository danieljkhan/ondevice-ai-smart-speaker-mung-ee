# llama-cpp-python 0.3.20 + llama.cpp b8772 for Jetson Orin Nano Super

This release publishes the Phase 0 `v0.3.20-llama-b8772` artifacts used by
Mungi's Gemma 4 E2B Q5_K_M text-only LLM path.

## Target hardware

- Device: Jetson Orin Nano Super
- Runtime: JetPack 6.2 / L4T R36.5.0 / Ubuntu 22.04
- Accelerator: CUDA 12.6 / aarch64 / sm_87
- Python: 3.10

## Upstream composition

- Python wrapper: `abetlen/llama-cpp-python` v0.3.20 at
  `02d6beed816535cb2ce845ad8558d0c8662f8e7e`.
- llama.cpp core: `ggml-org/llama.cpp` tag `b8772` at
  `bafae276541279aca751c25651462bc7b9134be3`.
- Build path: Plan v4 `scripts/build_llama_cpp.sh release` mode overrides the
  wrapper's vendored llama.cpp checkout to tag `b8772` before building.

## Build command

The release wheel was generated by `scripts/build_llama_cpp.sh` release mode
with the wrapper SHA pinned and the b8772 llama.cpp override enabled:

```bash
LLAMA_CPP_MODE=release \
LLAMA_CPP_PY_SHA=02d6beed816535cb2ce845ad8558d0c8662f8e7e \
LLAMA_CPP_BRANCH=b8772 \
bash scripts/build_llama_cpp.sh
```

The wheel CMake flags recorded in `release-provenance.json` are:

```text
-DGGML_CUDA=ON -DGGML_CUDA_F16=ON -DGGML_CUDA_FA_ALL_QUANTS=ON -DGGML_CUDA_PEER_MAX_BATCH_SIZE=512 -DGGML_NATIVE=OFF -DLLAVA_BUILD=OFF -DCMAKE_CUDA_ARCHITECTURES=87
```

## Install command

```bash
bash scripts/install_llama_cpp.sh --from-release-b8772
```

## Verification

Run these checks in a clean Jetson Python 3.10 virtual environment after
installing from the release:

```bash
python - <<'PY'
import llama_cpp

assert llama_cpp.__version__ == "0.3.20"
print(llama_cpp.__file__)
PY
```

Then verify a short Gemma 4 E2B Q5_K_M chat completion:

```bash
python - <<'PY'
from llama_cpp import Llama

llm = Llama(
    model_path="/opt/mungi/ai_models/gemma-4-E2B-it-Q5_K_M.gguf",
    n_gpu_layers=99,
    n_ctx=2048,
    verbose=False,
)
response = llm.create_chat_completion(
    messages=[{"role": "user", "content": "Say hello in one short sentence."}],
    max_tokens=40,
)
print(response["choices"][0]["message"]["content"])
PY
```

## Known good with

- Gemma 4 E2B Q5_K_M GGUF.
- Sherpa-ONNX Qwen3-ASR-0.6B INT8 on the text-only path.

## Not supported

- `mmproj` audio encoder loading is not supported by this release. See Plan
  v3.2 non-goals and the abandoned Path B history.
