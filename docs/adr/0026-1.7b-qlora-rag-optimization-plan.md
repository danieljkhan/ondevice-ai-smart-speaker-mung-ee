# ADR 0026: 1.7B QLoRA + KoEn-E5-Tiny RAG Optimization Plan

- **Status**: Proposed
- **Date**: 2026-03-26
- **Deciders**: User (Daniel) + Claude Code PM + Codex Reviewer

## Context

E2E 60-round RAG test (2026-03-26) revealed that the current RAG setup is not viable for production:

- **LLM (Qwen3-4B-Q4_K_M, 2.4GB)** + **E5-small embedding (449MB CPU)** = GPU layer downgrade (`n_gpu_layers` -1 → 20)
- LLM inference 3.8x slower with RAG active (3.22s → 12.24s)
- Turn time doubled (9.76s → 19.24s)
- SAFE_FALLBACK triggered 28x (vs 6x baseline) due to noisy wiki chunk injection

Root cause: E5-small (449MB) consumes too much memory on Jetson 8GB unified memory, forcing LLM to partial GPU offload.

## Decision

Replace the current LLM + embedding stack with a smaller, optimized combination:

### LLM: Qwen3-1.7B QLoRA Finetuned

| Item | Current | New |
|------|---------|-----|
| Model | Qwen3-4B-Q4_K_M (2.4GB) | Qwen3-1.7B QLoRA finetuned (~1.0GB Q4_K_M) |
| Training data | 7,710 samples | **20,000 samples** (2.6x increase) |
| Training categories | A-G | **A-I** (H: tone anchor, I: honorific correction) |
| Speed | 11.4 tok/s | ~17.4 tok/s (53% faster) |

Rationale: 1.7B model capacity is compensated by 2.6x more training data + 2 new specialized categories (H/I). Memory savings of ~1.4GB enables full GPU offload even with RAG active.

### Embedding: KoEn-E5-Tiny ARM64 qint8

| Item | Current | New |
|------|---------|-----|
| Model | intfloat/multilingual-e5-small (449MB) | exp-models/dragonkue-KoEn-E5-Tiny (38MB ONNX qint8) |
| Parameters | 118M | 37M |
| Korean NDCG@10 | 0.671 | **0.688 (+1.7pp better)** |
| Embedding dim | 384 | 384 (compatible) |
| Query prefix | "query: " | "query: " (identical) |
| ARM64 optimized | No | **Yes (model_qint8_arm64.onnx)** |

Rationale: Vocab-pruned Korean/English-only model. 1/3 parameters, better Korean benchmarks, ARM64 ONNX quantized available. Memory footprint reduced by 91%.

### Expected Memory Budget

| Component | Current (4B+E5) | New (1.7B+KoEn-E5-Tiny) |
|-----------|-----------------|-------------------------|
| LLM | 2.4GB | ~1.0GB |
| Embedding | 449MB | 38MB |
| FAISS + chunks | 100MB | 100MB |
| OS + Python | 1.5GB | 1.5GB |
| **Free for GPU** | **~4,873MB** | **~5,280MB** |
| `n_gpu_layers` | 20 (partial) | **-1 (full)** |

## Execution Plan

### Phase 1: Data Augmentation (20,000 samples)
1. Relax dedup threshold (0.85 → 0.92): ~12,000 samples recovered
2. Claude Batch API generation: ~16,150 raw samples (v5-1.7b preset)
3. Validate + deduplicate → final 20,000 samples
4. Categories A-I with H (tone anchor) and I (honorific correction) added

### Phase 2: KoEn-E5-Tiny RAG Rebuild + 4B A/B Test (Jetson)
1. Download KoEn-E5-Tiny ARM64 qint8 (38MB)
2. Rebuild FAISS index (43,234 chunks, same 384-dim)
3. Deploy to Jetson `/opt/mungi/ai_models/rag/`
4. **Test A: 4B finetuned + KoEn-E5-Tiny** — E2E 60-round RAG active test
5. Compare against existing 4B+E5-small baseline (2026-03-26 report)
6. Record: turn time, LLM time, n_gpu_layers, quality metrics

### Phase 3: 1.7B Finetuning (Colab A100)
1. Base: Qwen/Qwen3-1.7B
2. QLoRA: r=64, alpha=16, dropout=0.05
3. Data: 20,000 samples, epochs=5
4. Output: GGUF Q4_K_M + Q8_0

### Phase 4: 1.7B Quality Validation + A/B Test (Jetson)
1. Deploy 1.7B GGUF to Jetson
2. **Test B: 1.7B finetuned + KoEn-E5-Tiny** — E2E 60-round RAG active test
3. Compare against Test A (4B + KoEn-E5-Tiny)
4. Pass criteria: hallucination <10%, quality B+, honorific 0%, repetition <5%

### Phase 5: Final Decision
Based on Test A vs Test B comparison:

| Scenario | Decision |
|----------|----------|
| Test A good + Test B good | Choose 1.7B (faster, more memory headroom) |
| Test A good + Test B bad | Keep 4B + KoEn-E5-Tiny |
| Test A bad + Test B bad | Investigate further (chunk cleanup, context limit, BM25) |

**A/B Comparison Matrix**:

| Metric | 4B+E5-small (done) | 4B+KoEn-E5-Tiny (Test A) | 1.7B+KoEn-E5-Tiny (Test B) |
|--------|-------------------|--------------------------|----------------------------|
| n_gpu_layers | 20 | expected -1 | expected -1 |
| Avg LLM | 12.24s | expected ~3.2s | expected ~2.0s |
| Turn avg | 19.24s | expected ~10s | expected ~7s |
| Memory | 4,873MB | expected ~5,000MB | expected ~5,280MB |
| Quality | 8.2% repeat | TBD | TBD |

Target: turn avg <8s, repetition <5%, n_gpu_layers=-1

## Consequences

### Positive
- Full GPU offload restored with RAG active
- ~2x faster inference (1.7B vs 4B)
- 91% embedding model size reduction (449MB → 38MB)
- Better Korean retrieval quality (+1.7pp NDCG@10)
- More training data (20,000 vs 7,710) for robust finetuning

### Negative
- 1.7B has inherently lower reasoning capacity than 4B
- Requires FAISS index rebuild
- Quality risk until Phase 3 validation

### Neutral
- FAISS vector dimension unchanged (384)
- E5 query prefix identical ("query: ")
- Rollback path available (4B backup in `/opt/mungi/ai_models/backup/`)
