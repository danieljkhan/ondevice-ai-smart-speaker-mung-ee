## 1. Vision Statement

**"세상에서 가장 안전한, 우리 아이의 첫 번째 AI 친구"**

뭉이(Mung-i)는 10세 미만 아동을 위한 **완전 오프라인 한/영 이중언어 교육·말벗 AI 디바이스**입니다.

인터넷 없이 기기 내부에서 AI가 직접 실행되며, 아동의 음성·대화 데이터가 기기 밖으로 절대 전송되지 않습니다. 부모에게는 **세상에서 가장 안전하다는 신뢰**를, 아이에게는 언제나 곁에 있는 **첫 번째 AI 친구**를 제공합니다.

---

## 2. Problem Statement

### 아이들에게 AI가 필요한 이유

- 아이들은 끝없이 질문하고 대화하고 싶어하지만, 부모는 항상 곁에 있을 수 없습니다.
- 기존 AI 스피커(Alexa, Google Home 등)는 아동 전용으로 설계되지 않았고, 부적절한 콘텐츠 노출 위험이 있습니다.
- 클라우드 기반 AI는 아동의 음성 데이터를 외부 서버로 전송하며, 이는 COPPA/GDPR 등 아동 개인정보 보호 법규와 충돌합니다.

### 부모가 원하는 것

- **"내 아이의 신상정보가 완벽하게 보호된다"** 는 확신
- 아이가 혼자 놀 때도 안심할 수 있는 안전한 대화 상대
- 사용 내역을 투명하게 확인할 수 있는 보호자 통제 기능

---

## 3. Target Users

### Primary: 3~10세 아동

- 호기심이 많고 끊임없이 질문하는 나이
- 읽기/쓰기보다 **음성 상호작용**이 자연스러운 연령
- 디지털 기기에 대한 자체 판단력이 부족하여 안전 장치가 필수

### Secondary: 보호자 (부모/양육자)

- 아이의 디지털 기기 사용에 대한 통제와 가시성을 원함
- 대화 내역 열람, 사용 시간 제한, 야간 비활성화 등 관리 기능 필요
- 개인정보 보호에 민감하며, 데이터가 외부로 나가지 않는다는 보장을 원함

## 4. Core Value Proposition

| 가치 | 설명 |
|------|------|
| **Privacy-First** | 모든 데이터가 기기 내부에서만 처리되고, 외부 전송이 물리적으로 불가능 |
| **Child-Safe** | 3중 안전 필터(시스템 프롬프트 → 로컬 후처리 → 행동 제한)로 부적절 응답 차단 |
| **Offline-Only** | 인터넷 없이 완전 동작 — Wi-Fi 없는 차량, 야외에서도 사용 가능 |
| **Always Available** | 아이가 원할 때 즉시 대화 가능 — 클라우드 서버 장애나 네트워크 지연 없음 |
| **Transparent** | 보호자가 대화 내역을 로컬 웹 대시보드에서 직접 확인·관리 가능 |

---

## 5. Design Principles

1. **Privacy-First** — 모든 데이터는 기기 내부에서만 처리, 외부 전송 불가
2. **Child-Safe** — 3중 안전 필터로 부적절 응답 차단, 사용 시간/행동 제한
3. **Offline-Only** — 인터넷 미연결 상태에서 완전 동작, 클라우드 의존 없음
4. **Solo-Friendly** — 1인 개발자(직장인 퇴근 후/주말)가 구축 가능한 범위로 스코프 관리
5. **MVP-First** — 핵심 대화 기능으로 빠르게 Working Mockup 완성, 피드백으로 판단
6. **Open-Source First** — AI 모델 5종 전부 오픈소스, 직접 개발 최소화
7. **AI-Augmented Dev** — AI 코딩 에이전트 3종 활용하되, 안전/핵심 로직은 직접 작성

## 6. Product Architecture

### 6.1 Interaction Model

아이의 음성을 듣고, 이해하고, 생각하고, 대답하는 **4단계 음성 대화 파이프라인**:

```
[아이가 말함] → ① VAD (음성 감지) → ② STT (음성→텍스트)
             → ③ LLM (생각·응답 생성) → ④ TTS (텍스트→음성)
             → [뭉이가 대답함]
```

### 6.2 AI Pipeline

| 단계 | 모델 | 실행 환경 | 메모리 |
|------|------|-----------|--------|
| ① VAD | Silero VAD | CPU (상주) | ~30MB |
| ② STT | Qwen3-ASR (per ADR 0055) | GPU (순차 로딩) | per ADR 0055 |
| ③ LLM (primary) | Gemma 4 E2B Q5_K_M (llama.cpp, per ADR 0073) | GPU (순차 로딩) | ~2.0–2.5GB |
| ③ LLM (fallback) | Qwen3.5-2B-DPO Q6_K (env/config override) | GPU | ~2.0GB |
| ④ TTS | Supertonic TTS 2 (ONNX; sole TTS engine — Piper retired per ADR 0088) | CPU | ~0.5GB |
| ⑤ RAG | koen-e5-tiny ONNX + 별도 FAISS (대화 메모리, 100-token budget) | CPU/GPU | ~30–60MB |

### 6.3 Safety Architecture — 3중 필터

```
1차: 시스템 프롬프트 (아동 친화 지시 — "너는 10세 미만 아동의 친구 '뭉이'야")
2차: 로컬 후처리 (금칙어/패턴 차단 — 성적·폭력·자해 등)
3차: 행동 제한 (일일 사용 시간, 세션당 턴 수, 야간 비활성화)
```

### 6.4 Hardware Platform

- **NVIDIA Jetson Orin Nano Super** (8GB shared LPDDR5)
- NVMe 256GB SSD
- 3S Li-ion 배터리 + UPS BMS
- Waveshare Audio Card (마이크/스피커)
- PTT 버튼 (GPIO), 4inch HDMI 정전식 터치 디스플레이 (Waveshare 720×720, 표정 + 터치 입력)
- 패브릭 돔 외관 (수제작)

## 7. Success Metrics

### Phase 0 — HW + AI 검증 (현재)

| KPI | 목표 |
|-----|------|
| AI 모델 구동 | 5개 모델 전부 Jetson에서 개별 구동 |
| 시스템 피크 메모리 | ≤ 5.0GB (전체 시스템) |
| 외피 온도 (30분 연속) | ≤ 45°C |
| 아동 음성 STT CER | ≤ 30% (10명+ 샘플) |

### Phase 1 — MVP 코어

| KPI | 목표 |
|-----|------|
| E2E 응답 시간 (경로 A) | p50 ≤ 8초, p95 ≤ 10초 |
| 안전 필터 차단율 | ≥ 95% |
| 2시간 연속 안정성 | 메모리 증가 ≤ 200MB |
| 테스트 커버리지 | 40%+ |

### Phase 3 — Working Mockup 완성 판정

| KPI | 목표 |
|-----|------|
| 아동 만족도 | 3~5명 중 4명 이상 "또 하고 싶다" |
| 치명적 버그 | 0건 |
| Working Mockup 완성 기준 충족 | 전 항목 통과 |

## 8. Scope Definition

### In Scope (Working Mockup)

- 완전 작동하는 프로토타입 1대
- 아동이 실제로 사용 가능한 수준의 완성도
- 패브릭 돔 외관 (수제작 1대)
- 2시간 연속 안정 동작 검증
- 아동 사용자 테스트 3~5명
- 보호자 웹 대시보드 (Basic Auth)

### Out of Scope

- 양산 BOM 최적화
- KC 인증
- 크라우드펀딩
- 양산/판매 계획 (별도 계획서)

## 10. Competitive Differentiation

| 경쟁 제품 | 뭉이와의 차이 |
|-----------|---------------|
| Amazon Echo / Google Home | 클라우드 의존, 아동 전용 설계 아님, 데이터 외부 전송 |
| AI 챗봇 앱 (ChatGPT Kids 등) | 스크린 기반, 클라우드 의존, 아동 음성 데이터 외부 처리 |
| 기존 아동 완구 (스마트 인형 등) | AI 대화 불가 또는 단순 사전 녹음, 교육적 상호작용 제한 |

**뭉이의 핵심 차별점:**
1. **완전 오프라인** — 물리적으로 데이터 유출 불가
2. **실시간 AI 대화** — 사전 녹음이 아닌 진짜 AI 대화
3. **아동 전용 설계** — 처음부터 아동 안전을 중심으로 설계
4. **보호자 투명성** — 로컬 대시보드로 완전한 가시성 제공
5. **한/영 이중언어** — 한국어와 영어 모두 자연스러운 대화

## 11. Risk Summary

| 리스크 | 영향 | 완화 전략 |
|--------|------|-----------|
| 8GB 메모리 부족 | AI 모델 동시 로딩 불가 | 순차 로딩 전략으로 피크 ~3.6GB 유지 |
| E2E 응답 지연 | 아이 집중력 이탈 | 프리로딩 + Streaming TTS (Phase 2+) |
| 아동 음성 인식률 저하 | 대화 실패 | STT confidence 기반 재질문 UX |
| 배터리 안전 | 아동 안전 사고 | BMS 2중 보호 + 온도 모니터링 |
| 1인 개발 한계 | 일정 지연 | AI 에이전트 활용 + MVP 스코프 엄수 |
